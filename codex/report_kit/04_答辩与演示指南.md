# 答辩与演示指南

## 五分钟演示流程

### 1. 项目介绍，约 30 秒

> 本项目使用 C++17 自己实现了一个 LeNet 风格 CNN，用于 GTSRB 交通标志分类。Qt 只负责图形界面，CNN 数值计算没有调用 PyTorch、TensorFlow 或 Keras。

### 2. 网络结构，约 60 秒

结合结构图说明：

- Tensor 保存图片和特征图。
- 两层卷积提取局部特征。
- ReLU 提供非线性。
- MaxPooling 降低空间尺寸。
- 全连接层完成分类。
- Softmax 输出各类别概率。

### 3. 代码展示，约 90 秒

依次打开：

```text
source/src/cnn/Tensor.h
source/src/cnn/ConvLayer.cpp
source/src/cnn/CNN.cpp
source/src/cnn/Trainer.cpp
```

每个文件只讲一个重点，不要现场逐行阅读。

### 4. GUI 演示，约 90 秒

1. 双击 `application/run_demo.bat`。
2. 连续选择两到三种不同类别的演示图。
3. 展示 Top-1、置信度、Top-3 和推理时间。
4. 强调演示程序不需要重新训练。

### 5. 总结与限制，约 30 秒

说明当前稳定模型是 10 类 CPU 版本，完整 43 类训练和卷积加速属于后续改进方向。

## 常见问题

### 为什么叫“纯 C++ CNN”？

卷积、池化、ReLU、Flatten、全连接、Softmax、Loss 和反向传播均由项目源码实现，没有调用深度学习框架。

### 模型权重是什么？

权重是训练后学到的浮点参数。源码定义网络如何计算，`.bin` 文件保存网络已经学到的参数。

### 为什么使用 Qt？

Qt 只负责图片选择、界面显示和交互，不参与 CNN 计算。

### 为什么当前只有 10 类？

纯 CPU 朴素卷积训练完整 43 类耗时较长，因此稳定演示版采用可配置 10 类子集，同时保留扩展到 43 类的结构。

### 为什么准确率不是 100%？

交通标志存在光照、角度、尺度和类别相似性差异。课程项目重点是完整实现 CNN 流程，而不是依赖成熟框架追求工业级精度。

### 为什么模型不放进 Git？

机器学习权重和数据集属于大文件。源码由 Git 管理，权重通过 GitHub Release 分发，这是常见项目管理方式。

### Softmax 有什么作用？

它把网络最后的数值转换成总和为 1 的类别概率，便于选择 Top-1 和展示置信度。

### 如何扩展到 43 类？

准备完整 GTSRB，设置输出类别数为 43，使用 43 行标签文件重新训练即可。当前数据读取和动态输出层已保留这种能力。

## 演示前检查

- 提前完整解压资料包。
- 确认 `run_demo.bat` 能启动。
- 至少试运行三张演示图。
- 关闭无关窗口和通知。
- 准备 GUI 截图作为程序无法启动时的备用材料。
- 不要在答辩现场开始训练。
