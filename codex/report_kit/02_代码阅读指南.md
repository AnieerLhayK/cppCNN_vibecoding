# 代码阅读指南

## 推荐阅读顺序

```text
Tensor
  -> Layer
  -> Conv / ReLU / MaxPooling
  -> Flatten / Fully Connected
  -> Softmax / Loss
  -> CNN
  -> Trainer
  -> DataLoader / ImageProcessor
  -> CLI App
  -> Qt GUI Controller / QML
```

## 第一部分：数据怎么保存

从以下文件开始：

```text
source/src/cnn/Tensor.h
source/src/cnn/Tensor.cpp
```

`Tensor` 使用一维 `float` 数组保存 `channels x height x width` 数据，并提供维度检查和索引访问。图片、特征图、全连接输出都使用同一种结构。

## 第二部分：CNN 层

| 模块 | 文件 | 重点 |
| --- | --- | --- |
| 层接口 | `src/cnn/Layer.h` | 每层都提供 `forward` 和 `backward` |
| 卷积 | `ConvLayer.*` | 卷积前向、梯度和参数更新 |
| 激活 | `Activation.*` | ReLU 和 Softmax |
| 池化 | `PoolingLayer.*` | MaxPooling 与最大值位置缓存 |
| 展平 | `FlattenLayer.*` | 特征图变成一维向量 |
| 全连接 | `FCLayer.*` | 分类器权重和偏置 |
| 损失 | `Loss.*` | 交叉熵和准确率统计 |

## 第三部分：完整网络

阅读：

```text
source/src/cnn/CNN.h
source/src/cnn/CNN.cpp
```

当前稳定 LeNet 风格结构：

```text
输入 3x32x32
-> Conv(3->6, 5x5)
-> ReLU
-> MaxPool(2x2)
-> Conv(6->16, 5x5)
-> ReLU
-> MaxPool(2x2)
-> Flatten
-> FC(400->120)
-> ReLU
-> FC(120->类别数)
-> Softmax
```

`CNN.cpp` 还负责模型参数保存、加载和格式检查。

## 第四部分：训练流程

阅读：

```text
source/src/cnn/Trainer.h
source/src/cnn/Trainer.cpp
```

重点关注：

- mini-batch 训练循环
- 前向传播
- 交叉熵损失
- 反向传播
- SGD 更新
- 每轮 loss 和 accuracy 统计

## 第五部分：数据和图片

```text
source/src/data/DataLoader.*
source/src/image/ImageProcessor.*
```

`DataLoader` 根据类别目录生成图片路径和标签；`ImageProcessor` 负责读取、缩放到 `32x32` 并归一化到 `[0, 1]`。

## 第六部分：程序入口和 GUI

```text
source/src/main.cpp
source/src/app/App.*
source/src/gui/AppController.*
source/src/gui/InferenceEngine.*
source/qml/Main.qml
```

`App` 提供训练、评估和预测命令；`AppController` 连接 C++ 推理逻辑与 QML 界面；`Main.qml` 定义视觉布局和交互。

## 报告中适合展示的代码

建议截图或摘录：

1. `Tensor::at`：说明三维数据访问。
2. `ConvLayer::forward`：说明卷积计算。
3. `MaxPoolingLayer::forward`：说明降采样。
4. `CNN` 构造函数：说明网络结构。
5. `Trainer::train`：说明训练循环。
6. `DataLoader::loadDataset`：说明数据集读取。
7. `AppController::predict`：说明后台推理与 GUI。

不要在报告中堆放整页代码。每段代码后说明“输入、处理、输出和设计目的”。
