# 宝宝级资料包指南

这份指南给不熟悉 GitHub、CMake、Qt 或工程结构的同学使用。请先把 Release ZIP 完整解压，不要在压缩软件预览窗口里直接运行。

## 一、先运行演示程序

进入：

```text
application/
```

双击：

```text
run_demo.bat
```

程序启动后，可以点击下方演示图片，或按 `Ctrl+O` 选择自己的交通标志图片，再点击 `Recognize` 或按 `Ctrl+Enter`。界面会显示预测类别、置信度、Top-3 概率和推理耗时。

快捷键：

- `Ctrl+O`：选择图片。
- `Ctrl+Enter`：运行识别。
- `Esc`：清空当前图片。
- `Ctrl+M`：手动切换 `.bin` 模型。

## 二、每个目录是干什么的

```text
application/       # 可直接运行的 Windows 便携程序
source/            # Codex 版本源码，不含完整数据集和模型权重
report_materials/  # 报告、设计、训练结果、截图和 AI 使用记录
```

重要文件：

| 文件或目录 | 作用 | 能否删除 |
| --- | --- | --- |
| `application/cppcnn_gui.exe` | 图形界面程序 | 否 |
| `application/cppcnn_app.exe` | 命令行训练、评估、预测程序 | 否 |
| `application/models/gtsrb_v2_subset10.bin` | 默认 10 类演示模型 | 否 |
| `application/models/gtsrb_v5_full43.bin` | 完整 43 类 Enhanced 模型 | 否 |
| `application/models/*.labels.txt` | 模型对应类别名称 | 否 |
| `application/demo_images/` | 演示图片 | 建议保留 |
| `application/Qt6*.dll`、`plugins/`、`qml/` | Qt 运行资源 | 否 |
| `report_materials/codex_session_export.html` | AI 使用记录 | 必须提交 |

模型权重可以理解为 CNN 训练后学到的浮点参数。源码定义网络怎么计算，`.bin` 文件保存网络已经学到的识别能力。

## 三、演示图片

Release 中会打包若干张 GTSRB 测试图片，覆盖限速、禁止超车、优先通行、警告和强制方向等类别。演示时不需要展示太多，建议选 4 到 6 张不同类型图片即可。

报告资料放在：

## 五、报告需要强调的重点

### 1. 使用了 GitHub 团队开发

可以简单提到项目使用 GitHub 管理代码、分支、提交记录和 Release。不同实现路线放在不同目录中，`codex/` 是本次课程主交付，`claude/` 是早期对照原型。

不需要把 Git 命令讲得很细，重点是说明我们有版本管理、阶段提交和可下载交付包。

### 2. 主体部分用 C++ 验证 CNN 可以实现

项目核心不是调用现成深度学习框架，而是在 C++ 中实现 Tensor、卷积、池化、ReLU、Flatten、全连接、Softmax、Loss、训练统计和模型保存读取。

当前已经获得可用模型：

- `gtsrb_v2_subset10.bin`：默认 10 类便携演示模型。
- `gtsrb_v5_full43.bin`：完整 43 类 Enhanced 模型，可在 GUI 中通过 `Ctrl+M` 加载。

因此报告里可以明确说：C++ 实现不是空壳，已经跑通训练、导出、加载和推理演示。

### 3. 使用 Python/LibTorch 是为了加快训练

课程主线强调 C++ 实现 CNN。后期完整 43 类训练如果只靠朴素 CPU C++ 卷积会非常慢，因此加入了 LibTorch/CUDA 训练工具用于加速实验。

这个部分可以诚实表述为：训练加速工具服务于模型迭代和实验验证，最终演示 GUI 加载的是 `.bin` 模型，推理链路仍走项目自己的 C++ CNN 结构。

不要把项目说成“全程完全没有任何框架辅助”。更准确的说法是：

> CNN 核心和最终推理程序由 C++ 自己实现；为了在有限时间内完成完整 GTSRB 训练实验，开发阶段额外使用 GPU 加速工具生成并导出权重。

### 4. 项目从 MNIST 作业经验开始，后期复杂度显著提高

项目最开始可以参考以前 MNIST 作业的思路：图片输入、卷积网络、分类输出、准确率统计。

但 GTSRB 比 MNIST 复杂很多：

- 彩色 RGB 输入，不是灰度数字。
- 图片尺寸、亮度、角度差异更大。
- 类别数从简单数字扩展到交通标志。
- 数据集目录、标签、训练/测试划分更复杂。
- 还增加了 GUI、模型持久化、Release 打包和报告资料包。

后期仅靠手写和人工排查已经很难高效推进，因此大量依赖 AI 协助进行代码生成、调试、文档整理和打包自动化。这不是减分项，关键是要说明我们进行了验证、测试和阶段性收口，而不是盲目复制。

### 5. 模型迭代很明显

报告中适合展示模型演进：

| 阶段 | 模型 | 意义 |
| --- | --- | --- |
| v1 | LeNet plain | 原始基线 |
| v2 | LeNet + augmentation | 验证数据增强影响 |
| v3 | Enhanced plain | 提高网络容量 |
| v4 | Enhanced + augmentation | 最强 Phase B 基线 |
| v5 | Enhanced full43 | 80 epoch 正式训练，导出给 GUI 使用 |

这能说明项目不是一次性写完，而是从小模型、小样本、10 类演示逐步发展到完整 43 类模型。

### 6. 可以适度肯定项目亮点

表达可以朴素一点：

> 这个项目的价值不只在于最后的准确率，而在于把一个课程级 CNN 系统从数据集读取、图像预处理、网络层实现、训练评估、模型保存、GUI 展示到 Release 打包串成了完整流程。它不是工业级框架，但对本科 C++ 课程作业来说，已经覆盖了机器学习项目从实验到交付的大部分关键环节。

不要夸张成“工业级”“完全原创科研成果”或“超越成熟框架”。稳一点更可信。

## 六、答辩演示建议

五分钟流程：

1. 介绍目标：C++ CNN 交通标志识别。
2. 展示 GUI：运行 `run_demo.bat`，识别几张不同类别图片。
3. 讲核心代码：按 `02_代码阅读指南.md` 展示 Tensor、Conv、CNN、Trainer。
4. 讲模型迭代：LeNet 到 Enhanced，10 类到 43 类。
5. 讲限制：小类准确率仍低，GPU 训练工具与纯 C++ 推理链路需要区分。

常见问题回答：

- 为什么叫纯 C++ CNN？核心层和推理链路由 C++ 自己实现。
- 为什么又出现 LibTorch？为完整 43 类训练加速，并导出权重给 C++ GUI 使用。
- 为什么模型不进 Git？数据集和模型权重属于大文件，通过 Release 分发更规范。
- 为什么准确率不是 100%？GTSRB 类别不均衡，部分小类样本少，课程项目重点是完整流程。

## 七、不要做的事

- 不要删除 `application/models/`。
- 不要拆散 `application/` 里的 Qt DLL、`plugins/` 或 `qml/`。
- 不要把 `codex_session_export.html` 漏交。
- 不要把 `report_materials/project_report.md` 当成最终汇报稿直接复制。
- 不要声称完整 GTSRB 数据集已经被打进 Git 或 Release。
- 不要把 LibTorch 训练工具说成最终用户必须安装的依赖；普通演示只需要便携包。
