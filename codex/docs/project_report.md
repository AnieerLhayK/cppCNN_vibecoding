# 基于纯 C++ 的 CNN 交通标志识别系统项目报告

## 1. 选题背景

交通标志识别是智能驾驶与计算机视觉中的典型分类任务。成熟框架能快速建立模型，但隐藏了张量布局、卷积、梯度与参数更新。本项目面向本科 C++ 课程设计，从零实现小型 CNN，使完整计算过程可阅读、调试和展示。

仓库根目录用于比较不同 AI 平台的实现方式；除根 README 外，本实现的新增源码、文档和交付物均位于 `codex/`。

## 2. CNN 原理简述

卷积层通过局部共享卷积核提取边缘、颜色和形状特征；ReLU 提供非线性；最大池化降低空间尺寸；Flatten 将特征图展开；全连接层映射到类别空间；Softmax 输出和为 1 的概率。

训练使用交叉熵：

```text
L = -log(p_y)
```

反向传播按链式法则计算梯度，mini-batch SGD 更新卷积和全连接参数。测试准确率为 Top-1 正确数除以样本总数。

## 3. 系统设计

系统分为五层：

1. 数值核心：Tensor、Layer 与 CNN 层。
2. 数据层：图片预处理、GTSRB 扫描和标签映射。
3. 训练层：损失、反向传播、SGD、保存和评估。
4. 应用层：CLI 命令与资源检查。
5. 交互层：Qt Quick GUI、运行时资源发现、后台推理和便携部署。

核心层不依赖 Qt、OpenCV 或深度学习框架。GUI 通过 `ImageBridge` 将 `QImage` 转为核心 Tensor。

## 4. 数据集说明

GTSRB 共 43 类。本地已准备官方训练图 39,209 张、测试图 12,630 张及测试标签 CSV。开发阶段使用固定种子生成 10 类子集：

- 训练图：10,000 张，每类 1,000 张
- 测试图：5,670 张
- 输入：`32 x 32` RGB
- 归一化：`[0,1]`

完整数据集和子集不进入 Git，来源及目录见 `dataset_guide.md`。

## 5. 模型结构

```text
输入 3x32x32
Conv 3->6, 5x5      输出 6x28x28
ReLU
MaxPool 2x2         输出 6x14x14
Conv 6->16, 5x5     输出 16x10x10
ReLU
MaxPool 2x2         输出 16x5x5
Flatten             输出 400
FC 400->120
ReLU
FC 120->N
Softmax             输出 N 类概率
```

10 类模型参数量为 52,202。输出层由类别数构造，因此代码结构支持 43 类。

## 6. 核心代码模块

- `Tensor`：连续 CHW 数据和安全索引。
- `ConvLayer`：卷积前向、输入梯度、权重梯度和偏置梯度。
- `MaxPoolingLayer`：记录最大值位置用于反向传播。
- `ReLULayer` / `SoftmaxLayer`：激活与概率输出。
- `FCLayer`：矩阵向量计算和参数更新。
- `CrossEntropyLoss`：损失及输出梯度。
- `CNN`：网络拓扑、预测和自定义模型格式。
- `Trainer`：训练循环、损失和准确率统计。
- `DataLoader`：官方 GTSRB 与子集目录读取。
- `ResourceLocator`：统一查找开发目录和便携目录中的模型、标签与演示图。
- `InferenceEngine`：执行前向推理、Top-3 排序和耗时统计。
- `AppController`：协调模型状态、图片状态、异步任务与可读错误。
- `Main.qml`：深色仪表盘式交互界面。

## 8. 问题与改进

- 朴素 CPU 卷积导致完整 43 类训练较慢。
- 当前无数据增强、Adam、BatchNorm、验证集早停。
- GUI 暂不承担训练任务。
- 模型格式使用本机二进制字节序，主要面向 Windows/MSVC。
- Git 不保存训练权重，公开分发使用 GitHub Release 资产。

后续可增加 OpenMP/SIMD、im2col、数据增强、混淆矩阵、批量预测、训练进度页面和完整 43 类模型。

## 9. 总结

项目完成了不依赖深度学习框架的 C++ CNN 训练与推理闭环，支持真实 GTSRB、模型持久化、准确率评估、Qt 桌面 GUI 和便携 Windows 发布。它能够编译、训练、测试、演示，也保留了扩展到 43 类和更高性能实现的清晰路径。
