# 基于纯 C++ 的 CNN 交通标志识别系统项目报告

## 1. 选题背景

交通标志识别是智能驾驶和计算机视觉中的典型分类任务。成熟深度学习框架可以快速搭建模型，但会隐藏张量、卷积、梯度和参数更新的实现细节。本项目以课程设计为目标，使用 C++17 从零实现小型 CNN，使网络的前向传播、反向传播和训练过程均可阅读、调试和展示。

本仓库用于比较不同平台的 C++ CNN vibe coding 实现。按用户要求，除本项目 `codex/` 目录外，仅在仓库根目录新增了 `README.md`，用于解释 `codex/`、`claude/` 等一级目录的仓库定位；其余新增源码、文档和配置均位于 `codex/`。

## 2. CNN 原理简述

卷积层使用可学习卷积核在局部感受野上计算加权和，提取边缘、颜色和形状等特征。ReLU 将负值置零，增加非线性。最大池化在局部窗口取最大值，降低空间尺寸和计算量。Flatten 将特征图展开，全连接层完成类别映射，Softmax 将 logits 转换为和为 1 的类别概率。

训练使用交叉熵：

```text
L = -log(p_y)
```

网络根据链式法则从 Softmax 向前逐层计算梯度，再用小批量 SGD 更新卷积核、全连接权重和偏置。测试准确率定义为 Top-1 预测正确数除以样本总数。

## 3. 系统设计

系统分为四层：

1. 数值核心：Tensor、Layer 及各 CNN 层。
2. 数据层：图片预处理、数据集扫描和标签映射。
3. 应用层：训练、评估、模型保存/加载和预测。
4. 交互层：命令行菜单与可选 OpenCV 图片窗口。

核心层不依赖 OpenCV。OpenCV 未安装时，项目仍能使用 PPM 数据完成完整训练和推理。

## 4. 数据集说明

目标数据集为 GTSRB，共 43 类。本次从官方 ERDA 归档下载并核验了 39,209 张训练图、12,630 张测试图和官方测试 CSV。为兼顾课程演示时间，开发阶段使用自动生成的 10 类子集：每类 500 张训练图，共 5,000 张训练图和 5,670 张测试图。类别数由命令行和数据目录决定，输出层会动态构造，因此可直接切换到 43 类。

图片统一双线性缩放为 `32 x 32` RGB，并归一化至 `[0,1]`。完整数据集不进入仓库，下载和目录整理见 `dataset_guide.md`。

## 5. 模型结构

```text
输入 3x32x32
Conv 3->6, 5x5       输出 6x28x28
ReLU
MaxPool 2x2          输出 6x14x14
Conv 6->16, 5x5      输出 16x10x10
ReLU
MaxPool 2x2          输出 16x5x5
Flatten              输出 400
FC 400->120
ReLU
FC 120->N
Softmax              输出 N 类概率
```

该结构接近 LeNet，参数量和计算量适合本科课程展示。N 默认为 10，也可以设为 43。

## 6. 核心代码模块

- `Tensor`：连续 CHW 数据、尺寸信息、边界检查。
- `ConvLayer`：卷积前向、输入梯度、权重梯度和偏置梯度。
- `MaxPoolingLayer`：保存最大值索引，将梯度送回对应位置。
- `ReLULayer`：保存激活掩码。
- `FCLayer`：矩阵向量乘法和参数梯度。
- `SoftmaxLayer`：减最大值避免指数溢出，使用 Jacobian-vector product 反传。
- `CrossEntropyLoss`：损失与概率梯度。
- `CNN`：LeNet 拓扑、Top-1、模型持久化。
- `Trainer`：mini-batch SGD、L2 衰减、损失和准确率。
- `DataLoader`：数字类别目录、类别和样本限制、洗牌。
- `App` / `SimpleUI`：训练、评估、单图和交互预测。

### 7.2 自动测试

Release 配置编译成功，CTest 结果：


测试覆盖：

- Tensor 尺寸、索引和越界异常
- 图片双线性缩放和 `[0,1]` 归一化

### 7.3 端到端演示

项目提供两类合成 PPM 图案生成器，用于在没有 GTSRB 时验证命令链：

```text
生成数据 -> 训练 -> 保存模型 -> 测试集评估 -> 单图预测
```

实际使用 40 张合成训练图（每类 20 张）、12 张合成测试图（每类 6 张）、3 个 epoch 完成验证：

```text
Epoch 1: loss 0.7309, accuracy 47.50%
Epoch 2: loss 0.6388, accuracy 67.50%
Epoch 3: loss 0.4931, accuracy 80.00%
Test:    loss 0.4825, accuracy 100.00%
Single image: class 0, confidence 64.70%
```

训练损失持续下降，保存后的模型可重新加载并完成评估和预测。合成图案只有两类且差异明显，该结果只证明软件流程可运行，不应作为真实交通标志识别准确率。

### 7.4 GTSRB 实验状态

数据集和模型按照机器学习项目规范留在本地并被 Git 忽略。已完成完整 43 类结构烟雾测试：每类取 1 张训练图、训练 1 个 epoch，再从官方测试 CSV 每类取 1 张评估。该测试成功读取 43 张训练图和 43 张测试图，证明 43 类原始目录、CSV 标签、模型保存和评估链路可运行。由于样本和训练轮数极少，得到的 `2.33%` 准确率不代表模型能力。

## 8. 问题与改进

纯 C++ 朴素卷积使用多层循环，完整 43 类训练速度明显低于成熟框架。当前没有数据增强、动量、Adam、学习率衰减、批归一化和验证集早停，因此真实精度不是本项目重点。

可继续改进：

- 使用 OpenMP、SIMD 或 im2col 加速卷积；
- 加入数据增强和类别均衡采样；
- 增加动量 SGD/Adam 与学习率调度；
- 输出混淆矩阵、Top-5 和每类准确率；
- 增加更完整的 Windows GUI；
- 发布可复现的 10 类和 43 类模型。

## 9. 总结

项目完成了不依赖深度学习框架的 C++ CNN 核心、训练和推理闭环，支持 GTSRB 风格目录、模型持久化、测试集准确率、单图交互和 OpenCV 可选窗口。无 OpenCV、无真实数据集时仍有 PPM 合成数据保底演示，满足“能编译、能运行、能展示、能继续扩展”的课程设计目标。
