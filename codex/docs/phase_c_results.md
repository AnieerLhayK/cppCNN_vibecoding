## 5. 分析

### 架构能力

Enhanced 架构（3 Conv + 2 FC + Dropout, 1.29M 参数）在 43 类 GTSRB 上达到 **95.26% test accuracy**。相比 LeNet（56K 参数）的 87–89% 提升明显。但距离 98% 目标仍有差距，原因分析：

1. **无 BatchNorm**：当前实现无 BatchNorm 层，限制了深层网络的训练稳定性和泛化能力
2. **输入分辨率 32×32**：部分交通标志细节（如文字）在 32×32 下难以区分
3. **最低类问题**：类 0（20 km/h，仅 210 训练样本）和类 19（Dangerous curve left，210 样本）等极低样本类长期处于 50% 以下
4. **无额外正则化**：仅 weight decay + Dropout，对 1.29M 参数的网络可能不足

### 最佳 epoch 选择

Best checkpoint 在 epoch 30（97.5% val），此后 val acc 不再突破。实际 80 epoch 训练在 epoch 30 后无额外收益。建议未来设置 patience=10 的早停。

### 速度分析

| 阶段 | 耗时 |
|---|---|
| Epoch 1（CUDA JIT + 磁盘缓存） | 534.6s |
| Epochs 2–4（JIT 逐渐完成） | 149→91→68s |
| Epochs 5–80（稳定） | 49–75s/epoch |
| **总计** | **5330.9s（88.8 min）** |

稳定后平均 ~58s/epoch，扣除 JIT 和第一轮磁盘加载开销后的净训练时间约 68 分钟。

## 7. 模型对比与后续建议

| 模型 | 测试集 | 最低类 | 参数 | 结论 |
|---|---|---|---|---|
| LeNet（Phase B） | 87–89% | ~17–25% | 56K | 基线 |
| Enhanced（Phase C） | **95.26%** | **48.33%** | 1.29M | 当前最佳 |

### 后续改进方向

1. **添加 BatchNorm**：预计可将 test acc 提升至 97%+
2. **使用更大的输入（48×48 或 64×64）**：保留更多交通标志细节
3. **数据过采样**：对小类（样本 <300）进行重采样
4. **更长的训练（LR 调度优化）**：在 LR=0.001 阶段停留更长时间
5. **集成学习**：多个 checkpoint 投票
